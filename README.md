# Short Links App

Laravel-приложение для создания коротких ссылок, редиректа и просмотра статистики переходов.

Проект реализован в рамках тестового задания: пользователь может зарегистрироваться, создавать короткие ссылки, управлять ими в личном кабинете и просматривать статистику переходов.

## Стек

- PHP 8.3
- Laravel 10
- Filament v3
- MySQL 8
- Docker / Docker Compose

## Возможности

- регистрация и авторизация пользователей;
- личный кабинет на базе Filament v3;
- создание коротких ссылок;
- генерация уникального короткого кода;
- редирект по короткой ссылке на оригинальный URL;
- сохранение IP-адреса, User-Agent и даты перехода;
- просмотр общего количества кликов;
- просмотр истории переходов по каждой ссылке;
- удаление ссылок;
- ограничение доступа: пользователь видит только свои ссылки.

## Архитектура

Приложение состоит из двух основных частей.

### 1. Публичная часть

```text
GET /{code}
```

Публичный маршрут принимает короткий код, находит оригинальную ссылку, фиксирует переход и выполняет редирект на исходный URL.

Например:

```text
http://localhost:8080/abc123
```

При переходе по такой ссылке приложение:

- ищет запись в таблице `short_links` по полю `code`;
- сохраняет информацию о переходе в таблицу `link_clicks`;
- увеличивает счётчик `clicks_count`;
- перенаправляет пользователя на `original_url`.

### 2. Личный кабинет

```text
/cabinet
```

Личный кабинет реализован на Filament v3.

В кабинете пользователь может:

- зарегистрироваться;
- авторизоваться;
- создать короткую ссылку;
- посмотреть список своих ссылок;
- удалить ссылку;
- посмотреть статистику переходов по каждой ссылке.

## Основные сущности

### ShortLink

Модель короткой ссылки.

Основные поля:

- `user_id` — владелец ссылки;
- `original_url` — оригинальный URL;
- `code` — короткий уникальный код;
- `clicks_count` — общее количество переходов;
- `created_at` — дата создания;
- `deleted_at` — дата мягкого удаления.

### LinkClick

Модель перехода по короткой ссылке.

Основные поля:

- `short_link_id` — ссылка, по которой был переход;
- `ip_address` — IP-адрес пользователя;
- `user_agent` — информация о браузере/клиенте;
- `clicked_at` — дата и время перехода.

## Технические решения

- Генерация короткого кода вынесена в отдельный сервис `ShortCodeGenerator`.
- Создание короткой ссылки вынесено в сервис `ShortLinkCreator`.
- Фиксация перехода и увеличение счётчика вынесены в сервис `LinkClickRecorder`.
- Запись перехода и увеличение счётчика выполняются в транзакции.
- История переходов хранится отдельно от основной таблицы ссылок.
- `clicks_count` используется как быстрый счётчик общего количества переходов.
- Личный кабинет реализован через `ShortLinkResource` в Filament.
- История переходов отображается через Relation Manager.
- Пользователь видит только свои ссылки.
- Доступ к чужим ссылкам дополнительно ограничен через policy.

## Запуск проекта

Запустите Docker-контейнеры:

```bash
docker compose up -d --build
```

Установите зависимости:

```bash
docker compose exec app composer install
```

Создайте `.env` файл:

```bash
cp .env.example .env
```

Сгенерируйте ключ приложения:

```bash
docker compose exec app php artisan key:generate
```

Выполните миграции:

```bash
docker compose exec app php artisan migrate
```

Очистите кэш приложения:

```bash
docker compose exec app php artisan optimize:clear
```

## Адреса приложения

Главная страница:

```text
http://localhost:8080
```

Личный кабинет:

```text
http://localhost:8080/cabinet
```

Регистрация:

```text
http://localhost:8080/cabinet/register
```

Вход:

```text
http://localhost:8080/cabinet/login
```

## Создание пользователя через консоль

Если нужно быстро создать пользователя без формы регистрации, выполните:

```bash
docker compose exec app php artisan make:filament-user
```

После этого можно войти в личный кабинет через:

```text
http://localhost:8080/cabinet/login
```

## Пример сценария работы

1. Пользователь регистрируется в личном кабинете.
2. Создаёт короткую ссылку на оригинальный URL.
3. Получает короткую ссылку вида:

```text
http://localhost:8080/abc123
```

4. При переходе по короткой ссылке приложение:
    - находит оригинальный URL;
    - сохраняет IP-адрес, User-Agent и дату перехода;
    - увеличивает счётчик переходов;
    - перенаправляет пользователя на оригинальный URL.

5. В личном кабинете пользователь видит:
    - список своих ссылок;
    - количество переходов;
    - историю переходов по каждой ссылке.

## Docker

Проект запускается в Docker-среде и не требует локальной установки PHP, MySQL или Nginx.

Основные сервисы:

- `app` — PHP-FPM контейнер с Laravel-приложением;
- `nginx` — веб-сервер;
- `mysql` — база данных MySQL 8.

## Примечание

Проект реализован на Laravel 10 и Filament v3 в соответствии с требованиями тестового задания.

Для установки зависимостей в новых версиях Composer может потребоваться учитывать security advisory policy, так как Laravel 10 и Filament v3 относятся к более старым веткам. Настройки Composer зафиксированы в проекте.
